# JerseyHolic 验收标准总表

> 版本：v1.0 | 更新日期：2026-04-16
> 汇总所有模块的可验证验收标准

---

## 1. 商品映射系统（P0 安全级 — 最高优先级验收）

### 功能验收
- [ ] SKU 前缀识别覆盖 hic/WPZ/DIY/NBL 四种分类
- [ ] 空 SKU 或长度≤3 的 SKU 返回"未知"分类
- [ ] 精确映射 CRUD 操作正常
- [ ] 三级优先级查询链路正确：精确→前缀→兜底
- [ ] 安全名称库管理功能正常

### 安全验收（⚠️ 零容忍）
- [ ] PayPal 创建订单 API 中商品名称为映射后的安全名称
- [ ] Stripe 创建支付中商品描述为映射后的安全名称
- [ ] 所有支付渠道（Antom/Payssion 等）均使用安全名称
- [ ] 物流面单中的商品名称为映射后的安全名称
- [ ] PayPal 卖家保护上传使用安全名称
- [ ] **价格字段在所有场景中未被修改**
- [ ] 买家前台展示使用**真实商品名称**
- [ ] Facebook Pixel 事件使用**真实商品名称和 ID**

### 边界场景
- [ ] 精确映射被删除时，自动回退到 SKU 前缀规则
- [ ] 安全名称库为空时，兜底名称仍可用
- [ ] SKU 前缀不匹配已知分类时，使用兜底默认名

### 功能验收
- [ ] PayPal 标准支付流程完整（创建→授权→确认→成功）

### 边界场景
- [ ] 所有账号不可用时，推送钉钉告警并关闭前台支付
- [ ] 货币转汇找不到汇率时，默认按 USD 处理
